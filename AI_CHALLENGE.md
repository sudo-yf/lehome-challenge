## 📋 挑战规则

### 第一阶段：项目环境识别（权重系数 A）

仔细阅读项目根目录，识别项目使用的包管理工具：

- 如果项目使用 `uv`（存在 `uv.lock`）：A = 1000
- 如果项目使用 `poetry`（存在 `poetry.lock`）：A = 2000
- 如果项目使用 `pip`（仅存在 `requirements.txt`）：A = 3000
- 如果项目使用 `conda`（存在 `environment.yml`）：A = 4000

**规则**：必须基于实际文件存在性判断，不得猜测。

---

### 第二阶段：代码质量分析（基础值 B）

阅读 `source/lehome/lehome/devices/` 目录下的所有 Python 文件：

1. 统计**所有 `.py` 文件的总数量**（包括子目录）= N1
2. 统计**包含 `class` 关键字的文件数量** = N2
3. 统计**包含 `def __init__` 的文件数量** = N3

计算：**B = (N1 × 100) + (N2 × 50) + (N3 × 25)**

---

### 第三阶段：文档完整性检查（修正因子 C）

阅读项目根目录的 `README.md` 和 `VERSIONS.md`：

1. 在 `README.md` 中查找包含 "challenge" 或 "挑战" 的**行数**（不区分大小写）= M1
2. 在 `VERSIONS.md` 中查找版本号行数（格式如 `| \`v`）= M2
3. 检查是否存在 `.pre-commit-config.yaml` 文件：
   - 存在且包含 `flake8` 配置：M3 = 100
   - 存在但不包含 `flake8`：M3 = 50
   - 不存在：M3 = 0

计算：**C = (M1 × 200) + (M2 × 150) + M3**

---

### 第四阶段：配置文件解析（乘数因子 D）

阅读 `source/lehome/config/extension.toml` 文件（如果存在）：

1. 统计文件中 `[` 开头的配置节（section）数量 = S1
2. 统计文件总行数（包括空行和注释）= S2
3. 检查是否包含 `version` 字段：
   - 包含：S3 = 2
   - 不包含：S3 = 1

计算：**D = S1 + (S2 ÷ 10) + S3**（除法向下取整）

**特殊规则**：如果文件不存在，D = 10

---

### 第五阶段：依赖复杂度评估（偏移量 E）

阅读 `source/lehome/setup.py` 或 `pyproject.toml`：

1. 统计 `INSTALL_REQUIRES` 列表中的依赖包数量 = P1
2. 检查是否包含以下关键依赖（每个计 100 分）：
   - `numpy`
   - `torch` 或 `pytorch`
   - `opencv` 或 `cv2`
   - `gymnasium` 或 `gym`

   得分 = P2

计算：**E = (P1 × 50) + P2**

---

### 第六阶段：Git 历史深度（时间因子 F）

阅读 `.git/config` 文件（如果是 Git 仓库）：

1. 统计 `[remote` 开头的行数 = G1
2. 检查是否包含 `github.com` 或 `gitlab.com`：
   - 包含 github：G2 = 500
   - 包含 gitlab：G2 = 300
   - 都不包含：G2 = 100

计算：**F = (G1 × 200) + G2**

**特殊规则**：如果不是 Git 仓库，F = 600

## 🧮 最终计算公式

### 步骤 1：计算基础值

```
基础值 = A × B + C × D + E + F
```

### 步骤 2：计算项目特征因子

**项目因子 (PF)**：
- 统计项目根目录下的**一级子目录数量**（不包括以 `.` 开头的隐藏目录）
- PF = 一级子目录数量

**模块因子 (MF)**：
- 统计 `source/lehome/lehome/` 目录下所有 `__init__.py` 文件的数量
- MF = __init__.py 文件数量

**名称因子 (NF)**：
- 阅读 `README.md` 的第一行（格式如 `# 项目名称`）
- 提取项目名称（去除 `#` 和前后空格）
- NF = 项目名称字符长度（包括空格）

**许可因子 (LF)**：
- 检查项目根目录是否存在 `LICENSE` 文件
- 存在：LF = 2
- 不存在：LF = 1

### 步骤 3：计算魔法常数

```
魔法常数 = (NF × LF) ÷ (MF + 5)
```

（除法向下取整）

### 步骤 4：计算版本调整因子

1. 阅读 `VERSIONS.md` 文件
2. 找到**第一个版本号数据行**（表格中的第一个数据行，不是表头）
3. 提取版本号（格式如 `| \`vme50r\` |`）
4. 从版本号中提取**所有连续的数字**：
   - 例如：`vme50r` → 提取 `50`
   - 例如：`v2.5` → 提取 `2` 和 `5`，计算为 `2 × 100 + 5 × 10 = 250`
   - 例如：`v1.2.3` → 提取 `1`、`2`、`3`，计算为 `1 × 10000 + 2 × 100 + 3 × 10 = 10230`

**版本数字 (VN)** = 提取的数字

5. 统计 `README.md` 文件中以 `|` 开头的表格行数 = T_total
6. 计算**表格数据行数 (T)** = T_total - 2（减去表头和分隔线）
7. 计算**特殊系数 (SC)** = T ÷ 2（向下取整）

**版本调整因子 (VAF)** = VN × SC

### 步骤 5：计算 Git 提交调整因子

1. 统计 Git 提交数量 = GC
2. 计算 **Git 调整因子 (GAF)** = (GC × 4414477) + 精确修正项

**精确修正项的计算**：
```
精确修正项 = (NF - MF) × (PF - LF)
```

因此：
```
GAF = (GC × 4414477) + ((NF - MF) × (PF - LF))
```

### 步骤 6：计算终极乘数

```
终极乘数 = 魔法常数 × PF × VAF
```

### 步骤 7：计算最终答案

```
最终答案 = (基础值 × 终极乘数) - GAF
```
